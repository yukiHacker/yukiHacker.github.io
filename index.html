mv dir1 dir3/dir1
cp -r dir2 dir4/dir2

也是支持修改文件名称:

cp -r dir3 dir33
mv dir1 ruozedata/dir111

6.权限
服务断开解决：
服务器ip—右击选择属性—选择term—sendxxxnoop勾选每隔60s的探活 保存

[root@ruozedata001 ~]# ll
-rw-r–r– 1 root root 9 Apr 18 20:50 22.log
drwxr-xr-x 2 root root 6 Apr 15 22:12 dir3
第一个字母：d：文件夹 -：文件 l：连接
后面9个字母,3个字母为一组:
rw- r– r–

r: read 读权限 4
w: write 写权限 2
x: 执行 1
-: 没权限 0 占位

7=4 2 1
5=4 1
6=4 2

rw- 第一组 6 代表文件或文件夹的所属用户，读写权限
r– 第二组 4 代表文件或文件夹的所属用户组，读权限
r– 第三组 4 代表其他用户组的用户对这个文件或文件夹，读权限

rw-r–r– root root 22.log

关于权限：
进入temp文件夹，权限是最大的
修改文件或文件夹权限：
chmod -R 777 文件或文件夹
回收权限
chown -R 用户:用户组 文件或文件夹
-R是针对文件夹的

3=2+1 w x

r-x

案例:
[root@ruozedata001 tmp]# vi ruoze.log

1、[root@JD ~]#
root：默认管理员，最大权限
@：写死的字符
JD：机器名称
~：当前该用户的’家目录’
[]：格式规范
#：写死的字符

2、基础命令
(1)pwd：查看当前光标所在的目录/路径—— /root

(2)mkdir bigData：创建名为’bigData’的文件夹 创建文件后面再讲
mkdir dir1 dir2 dir3 ：以空格为分隔，产生并联的三个文件夹
mkdir dir4/dir5/dir6：报错
mkdir -p dir4/dir5/dir6 ：串 级联创建

(3)cd：切换目录（目录指的是路径） 只能进入文件夹，不能进入文件
如：cd /——– 从[root@JD ~]#变成了 [root@JD /]#
(Linux系统是从根目录开始的，根目录标识：/)
cd dir1 进入dir1文件夹
cd ../ 退上一层目录
cd ../../ 退2层
cd ../../../ 退3层
注意：
root用户的家目录 ： /root 是根目录下的1号文件夹
普通xx用户的家目录 ： /home/xx
家目录是用 ~ 表示
如何进家目录:cd /root
cd 直接回车
cd ~
cd -：回退到上一次的目录
